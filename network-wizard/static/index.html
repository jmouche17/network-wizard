Using files in scripts
# Python scripts — use the injected 'files' helper:
content = files.read('commands.txt')      # full file as string
lines   = files.lines('commands.txt')     # list of non-empty lines
path    = files.path('template.j2')       # absolute path on disk
exists  = files.exists('commands.txt')    # check before reading

# If you selected a file in the Run Scripts dropdown:
content = open(selected_file, 'rb').read()   # selected_file = full path
lines   = files.lines(selected_filename)     # selected_filename = just the name

# Bash scripts — use the $FILES_DIR environment variable:
# cat "$FILES_DIR/commands.txt" | while read cmd; do
#     ssh $DEVICE_USERNAME@$DEVICE_IP "$cmd"
# done
# Or use the selected file directly: cat "$SELECTED_FILE"

Scheduler
Run scripts automatically on a schedule. Uses cron syntax — e.g. 0 2 * * * = 2am daily, */30 * * * * = every 30 minutes.
Cron quick reference
┌─────── minute (0-59)
│ ┌───── hour (0-23)
│ │ ┌─── day of month (1-31)
│ │ │ ┌─ month (1-12)
│ │ │ │ ┌ day of week (0-6, Sun=0)
│ │ │ │ │
* * * * *

0 2 * * *      every day at 2:00am
*/30 * * * *   every 30 minutes
0 * * * *      every hour
0 8 * * 1      every Monday at 8:00am
0 0 * * 0      every Sunday at midnight

Inbound webhook usage
# Run a script via webhook:
curl -X POST https://yourserver/api/webhooks/receive/YOUR-TOKEN \
  -H "Content-Type: application/json" \
  -d '{"action": "run_script", "script": "backup_config.py", "target": "core-sw-01"}'

# Check a device:
curl -X POST https://yourserver/api/webhooks/receive/YOUR-TOKEN \
  -d '{"action": "check_device", "device": "core-sw-01"}'

# Add a note to a device:
curl -X POST https://yourserver/api/webhooks/receive/YOUR-TOKEN \
  -d '{"action": "add_note", "device": "core-sw-01", "message": "Deployed new firmware"}'
